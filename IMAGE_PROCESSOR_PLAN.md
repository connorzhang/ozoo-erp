# Ozon 突破复制禁令：自建图床与自动洗稿方案规划

## 1. 背景与目标
在 Ozon 平台上，直接复制其他卖家的主图会触发“违反复制禁令”（Нарушение запрета на копирование），导致商品被下架。
本方案旨在通过 ERP 系统实现**全自动的图片去重（洗稿）**，并利用**自建图床**向 Ozon 提交全新图片，彻底解决跟卖/上新时的图片版权限制，且**无需额外购买任何第三方云存储（OSS）**。

## 2. 核心技术原理

### 2.1 自建图床 (Self-hosted Image Server)
- **原理**：Ozon 的 `/v1/product/pictures/import` 接口仅接受公网 URL。我们将利用 Go 语言内置的高并发 HTTP 服务器，开放一个 `/public/uploads/` 静态资源目录。
- **效果**：ERP 服务器直接兼任图床，Ozon 爬虫通过该目录的 URL 拉取图片，实现零成本存储。

### 2.2 深度洗稿引擎 (Image Processor)
- **原理**：Ozon 机器人通过检测文件的 MD5/SHA 哈希值和 EXIF 元数据来判断图片是否重复。
- **对策**：
  1. **元数据剥离**：重新编码图片时，自动丢弃原始 EXIF 信息。
  2. **像素微调**：将原图放入一个宽高各增加 2-4 像素的纯白背景画布中（或者做1%的微裁剪/加水印）。
- **效果**：肉眼视觉无明显差异，但在机器算法眼中是一张 100% 全网首发的全新图片。

### 2.3 阅后即焚机制 (Auto Cleanup)
- **原理**：防止长期运行导致服务器硬盘塞满临时图片。
- **对策**：编写 Go 后台定时任务（Cron/Goroutine），每小时自动扫描并彻底删除创建时间超过 2 小时的图片。

---

## 3. 执行步骤拆解

### 步骤 1：【后端基建】开放图床静态路由
- 在 `internal/app/server.go` 中添加静态文件服务。
- 确保 `uploads/` 文件夹在系统启动时自动创建。
- 组装可供外网访问的 URL（需结合服务器实际配置的公网域名或 IP）。

### 步骤 2：【核心算法】完善图片洗稿引擎
- 编写 `ProcessImage` 核心函数。
- 实现：传入外部图片 URL -> 内存下载 -> 图像解码 -> 增加 2px 白边框 -> 重新进行 JPEG/PNG 编码去 EXIF -> 以 UUID 为文件名保存至 `uploads/` 目录。

### 步骤 3：【API 对接】打通 Ozon 图片上传接口
- 在 `internal/app/server.go` 或专用服务中，对接 Ozon 的 `/v1/product/pictures/import` 接口。
- 将自建图床生成的公网 URL（如 `https://erp.yourdomain.com/public/uploads/uuid.jpg`）发送给 Ozon，并处理 Ozon 的异步拉取状态。

### 步骤 4：【系统清理】实现定时清道夫
- 编写 `internal/app/cleanup.go`。
- 在服务启动时开启一个 Goroutine，利用 `time.Ticker` 每小时执行一次清理。
- 遍历 `uploads/` 目录，检查文件 `ModTime`，删除超时的临时文件。

### 步骤 5：【前端集成】UI 界面联调
- 在前端 Web 界面中添加“突破禁令/图片洗稿”操作面板。
- 允许用户粘贴 1688 / 淘宝 / Ozon 竞品链接，前端将链接发给后端，后端走完上述 1-4 步，前台实时展示洗稿后并成功上传 Ozon 的新图。

- 后续开发“一键搬家/跟卖”功能时，可以直接调用此模块，实现 100% 绕过 Ozon 原创保护的安全上新。
